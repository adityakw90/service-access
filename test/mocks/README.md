# Test Mocks

This directory contains generated test mocks for all ports defined in `internal/core/port/`.

## Structure

Mocks are organized by port type, mirroring the structure of `internal/core/port/`:

```
test/mocks/
├── repository/     # Repository port mocks
│   ├── mock_group_repository.go
│   ├── mock_permission_repository.go
│   ├── mock_role_repository.go
│   ├── mock_subject_repository.go
│   ├── mock_repository_provider.go
│   └── mock_unit_of_work.go
├── security/       # Security port mocks
│   └── mock_uid_generator.go
├── resolver/       # Resolver port mocks
│   ├── mock_group_resolver.go
│   ├── mock_permission_resolver.go
│   ├── mock_role_resolver.go
│   ├── mock_group_permission_resolver.go
│   └── mock_resolver_provider.go
├── observer/       # Observer port mocks (if applicable)
├── event/          # Event port mocks
│   └── mock_event_publisher.go
└── service/        # Service port mocks
    ├── mock_access_service.go
    ├── mock_group_service.go
    ├── mock_permission_service.go
    ├── mock_role_service.go
    └── mock_subject_service.go
```

## Regenerating Mocks

After modifying any interface in `internal/core/port/`, regenerate the corresponding mock:

### All Mocks

```bash
make mocks
```

### Specific Port Type

```bash
mockery --config .mockery.yaml
```

The `.mockery.yaml` configuration file ensures that:
- All mocks use `package mocks`
- Mock names follow the `Mock{{InterfaceName}}` pattern
- Mocks are output to the appropriate subdirectory

## Usage

Import the mocks package and use in your tests:

```go
import testmocks "github.com/adityakw90/service-access/test/mocks"

func TestSomething(t *testing.T) {
    ctrl := gomock.NewController(t)
    defer ctrl.Finish()

    mockRepo := testmocks.NewMockGroupRepository(ctrl)
    mockRepo.EXPECT().
        FindByID(gomock.Any(), uid).
        Return(group, nil)

    // Use mock in test
}
```

## Conventions

1. **Do Not Edit Generated Files Directly** - All mocks are generated by mockery. Any manual changes will be overwritten on regeneration.

2. **Package Name** - All mocks use `package mocks` for consistent importing.

3. **Naming Pattern** - Mocks follow the `Mock{{InterfaceName}}` pattern (e.g., `GroupRepository` → `MockGroupRepository`).

4. **With Expecter** - All mocks are generated with `with-expecter: true` for gomock expectation setup.
